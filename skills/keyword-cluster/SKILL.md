Cluster these keywords by search intent and content type: $ARGUMENTS

## Instructions

If $ARGUMENTS is empty or unclear, ask: "Please paste your keyword list (comma-separated or one per line)."

### Step 1 — Classify each keyword by intent

| Intent | Definition | Signal words |
|---|---|---|
| **Informational** | User wants to learn | how, what, why, guide, tips, tutorial, examples |
| **Navigational** | User wants a specific site/brand | brand names, "login", "official" |
| **Commercial** | User is researching before buying | best, top, vs, review, compare, alternatives |
| **Transactional** | User wants to buy or sign up | buy, price, cheap, discount, coupon, download, free trial |

### Step 2 — Group by topic cluster

Keywords with the same core topic belong together. One keyword per cluster becomes the **pillar** (highest volume / most general). The rest are **supporting** keywords for that page or for cluster pages.

### Step 3 — Map to content type

- Informational → Blog post, guide, how-to article, FAQ
- Navigational → Homepage, brand page, login page
- Commercial → Comparison page, listicle ("Best X"), review post
- Transactional → Landing page, product page, pricing page

### Output format

For each cluster:

---
### [Cluster Name]
**Pillar keyword:** `[primary keyword]`
**Intent:** Informational / Commercial / Transactional / Navigational
**Recommended content type:** [e.g. "Long-form guide (1500+ words)"]
**Supporting keywords:**
- `keyword 2` — [brief note on angle or subtopic]
- `keyword 3` — [brief note]

**Content brief hint:** [One sentence on what this page should cover to rank for this cluster]

---

After all clusters, add a **Content gap summary**: keywords that don't fit existing clusters and may represent new content opportunities.
